1 fetching database from container-registry.oracle.com

at first we have to pull the container image from Oracles website for that use this: podman pull container-registry.oracle.com/database/express:21.3.0-xe
With this command you can pull the image file of database for installation

Starting installation in podman:

now this is the stage from where main work will begin!
for install the database use this: podman run -d --name oracle-xe -p 1521:1521 container-registry.oracle.com/database/express:21.3.0-xespan>
Explanation of options:


  • -d: This runs the container in detached mode, allowing the container to run in the background.

  • --name oracle-xe: This assigns a name (oracle-xe) to the container for easier identification and management.

  • -p 1521:1521: This maps the container's port 1521 (the default Oracle database port) to your host machine's port 1521. This allows you to connect to the database from your host machine.

2. Connect to the Database:


There are two main tools you can use to connect to the database and work with PL/SQL:
SQL*Plus:
This is a command-line tool included with the Oracle database installation. You might need to install it separately depending on your operating system. Here's how to connect with SQL*Plus:

  1. Open a terminal window.

  2. Type sqlplus and press Enter.

  3. When prompted for username, enter sys (default administrator account).

  4. When prompted for password, enter the password you set during the database installation (default password for XE is usually oracle).
